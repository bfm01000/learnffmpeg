## 3. 项目优先级

不要同时开太多坑，严格按这个顺序推进。

### 项目 1：视频前 N 帧导出工具

目标：

- 输入 MP4
- 解封装 + 解码
- 导出前 N 帧
- 支持保存为 YUV / PPM / JPEG
- 打印每帧的基本信息

必须吃透的点：

- `AVFormatContext`
- `AVCodecContext`
- `AVPacket` 与 `AVFrame`
- `pix_fmt`
- `frame->data[]`
- `frame->linesize[]`
- YUV420P / NV12 的内存布局

交付标准：

- 能稳定导出指定帧数
- 能解释为什么保存 YUV 时要按 `linesize` 逐行写
- 能清楚说明“Packet 是压缩数据，Frame 是原始数据”

### 项目 2：MP4 -> TS Remux 工具

目标：

- 不重新编码
- 只做流拷贝和封装转换
- 输入 MP4，输出 TS

必须吃透的点：

- remux 与 transcoding 的区别
- `avcodec_parameters_copy`
- `codec_tag = 0`
- `av_packet_rescale_ts`
- `time_base`
- `av_interleaved_write_frame`

交付标准：

- 能清楚说明为什么 remux 不需要解码和编码
- 能解释为什么时间戳必须重映射
- 能解释 MP4 和 TS 只是容器不同，不代表编码格式一定变了

### 项目 3：MiniTranscoder 简化版转码器

这是接下来最重要的项目，也是简历里的主项目。

目标：

- 输入 MP4 / MKV
- 输出 MP4
- 视频链路：decode -> scale/pix_fmt convert -> encode
- 音频链路：decode -> resample -> encode
- 最终 mux 成一个可正常播放的新文件

必须吃透的点：

- 视频转码完整链路
- 音频转码完整链路
- `libswscale`
- `libswresample`
- 编码器 flush
- 音视频同步
- PTS / DTS / `time_base` 换算

交付标准：

- 输出文件可播放
- 音画基本同步
- 能解释整个数据流转过程
- 能把项目讲成简历项目

## 6. 当前阶段最重要的学习原则

### 原则 1：不要继续散点式学习

后面再遇到新的知识点，优先判断：

- 它是不是当前项目必须要用？
- 它是不是高频面试题？
- 它是不是会影响我真正做出项目？

如果都不是，可以先放下。

### 原则 2：优先形成“完整链路意识”

先把链路打通，再追求细节极致。

优先顺序应该是：

- 先跑通
- 再理解
- 再优化
- 再工程化

### 原则 3：项目比零散知识点更重要

找工作时，真正能证明能力的不是“我看过这些资料”，而是：

- 我写过什么
- 我解决过什么问题
- 我能不能把原理讲清楚
