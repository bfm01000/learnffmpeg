## 参考答案（含讲解）

### 1. 结构差异（示例）

| 维度 | MP4 | TS |
|---|---|---|
| 数据组织方式 | 层级 box 结构（`moov`/`trak`/`mdat`） | 固定 188 字节包线性排列 |
| 元数据位置 | 主要集中在 `moov`（轨道信息在 `trak`） | 分散在 PAT/PMT 和 PES 头 |
| 媒体数据位置 | 主要在 `mdat`，由 sample 索引定位 | 分散在各 PID 的 TS 包 payload |
| 随机访问 | 借助索引表（如 stco/stsz/stts）更直接 | 通常依赖外部索引/分片，原生不如 MP4 方便 |
| 传输容错 | 更偏“文件存储”语义 | 设计上更偏传输，便于丢包检测与同步恢复 |

**讲解：**

- MP4 的核心是“索引驱动读取”，即先读元数据再按偏移取数据；
- TS 的核心是“流式分包复用”，即边收边解，多路数据靠 PID 识别；
- 因为组织方式不同，它们在播放控制能力和传输鲁棒性上的优势也不同。

### 2. 适用场景（示例）

- **MP4：**
  1. 点播视频平台（VOD）文件存储：需要良好随机访问、拖动进度和多轨管理。  
  2. 本地文件播放/下载后播放：单文件封装完整、兼容性高、管理方便。  
  3. 短视频发布与编辑中间文件：box 结构清晰，元数据表达能力强。

- **TS：**
  1. 直播分发（如 IPTV、广播）：固定包结构和 PID 机制便于持续传输。  
  2. HLS 传统切片（`.ts` 分片）：与直播链路兼容好，分片生成与分发成熟。  
  3. 不稳定网络链路下的实时媒体传输：对同步与流式处理更友好。

**讲解：**

- 如果业务主要是“文件管理 + 任意拖动 + 离线分发”，MP4 通常更合适；
- 如果业务主要是“实时传输 + 持续输出 + 广播级流程”，TS 通常更合适。

### 3. 总结（示例）

当需求偏向点播、文件存储和随机访问时，优先选择 MP4；它依赖 `moov`/`trak` 建立清晰索引，读取效率和兼容性更好。  
当需求偏向直播、广播或流式传输时，优先选择 TS；其固定 188 字节包和 PID 复用机制更适合实时链路。  
工程上经常二者配合：生产链路用 TS 传输，归档或点播侧转为 MP4。
